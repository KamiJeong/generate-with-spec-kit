# Feature Specification: SFood 브랜드 사이트

**Feature Branch**: `019-sfood-brand-site`  
**Created**: 2026-04-15  
**Status**: Draft  
**Input**: 새로운 브랜드 사이트 설계 — SFood(에쓰푸드) 프리미엄 육가공 식품 전문 기업 브랜드 사이트

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 메인 페이지 방문 및 브랜드 첫인상 (Priority: P1)

처음 사이트를 방문한 일반 소비자 또는 비즈니스 파트너가 SFood의 브랜드 아이덴티티와 핵심 메시지를 즉시 인식하고, 원하는 섹션으로 자연스럽게 이동할 수 있다.

**Why this priority**: 메인 페이지는 모든 방문자의 첫 접점이며, 브랜드 신뢰도와 방문자 이탈율에 가장 직접적인 영향을 미친다.

**Independent Test**: 메인 페이지(`/`) 단독 접속으로, 브랜드명·미션 문구·핵심 제품 카테고리·내비게이션 메뉴가 정상 표시되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 방문자가 사이트 루트 URL에 접속했을 때, **When** 페이지가 로드되면, **Then** 히어로 섹션에 SFood 브랜드명과 미션 슬로건("더 좋은 식품으로 더 좋은 세상을")이 표시된다.
2. **Given** 방문자가 메인 페이지를 스크롤할 때, **When** 각 섹션을 지나면, **Then** 브랜드 소개·대표 제품·수상 이력·지속 가능성 한줄 소개가 순서대로 노출된다.
3. **Given** 방문자가 내비게이션 메뉴를 클릭했을 때, **When** 메뉴 항목을 선택하면, **Then** 해당 페이지로 이동한다.

---

### User Story 2 - 회사 소개 및 브랜드 스토리 탐색 (Priority: P2)

방문자가 `/about` 페이지에서 SFood의 역사, 미션·비전, 품질 인증 정보를 확인하고 브랜드 신뢰를 형성할 수 있다.

**Why this priority**: 신규 비즈니스 파트너나 소비자가 브랜드 결정 전에 가장 많이 확인하는 페이지로, 신뢰 구축에 핵심적이다.

**Independent Test**: `/about` 접속만으로 회사 설립 배경·미션·품질 인증(FSSC 22000, HACCP, DLG 수상) 정보가 모두 확인 가능하다.

**Acceptance Scenarios**:

1. **Given** 방문자가 `/about`에 접속했을 때, **When** 페이지가 로드되면, **Then** 회사 미션("더 좋은 식품으로 더 좋은 세상을"), 핵심 가치, 연혁이 표시된다.
2. **Given** 방문자가 품질 인증 섹션을 확인할 때, **When** 해당 영역을 스크롤하면, **Then** FSSC 22000, HACCP, DLG 국제 품평회 수상 배지 및 설명이 노출된다.

---

### User Story 3 - 브랜드 라인업 탐색 (Priority: P2)

방문자가 `/brands` 페이지에서 SFood의 B2B·B2C 브랜드 포트폴리오(예: 존쿡 델리미트 등)를 한눈에 파악하고 각 브랜드의 특색을 이해할 수 있다.

**Why this priority**: 제품 구매 또는 파트너십 검토 시 브랜드 구조 파악이 필수적이며, 소비자·바이어 모두가 사용하는 핵심 정보 페이지다.

**Independent Test**: `/brands` 단독 접속으로 B2B/B2C 브랜드 카드가 표시되고 각 브랜드 설명을 읽을 수 있다.

**Acceptance Scenarios**:

1. **Given** 방문자가 `/brands`에 접속했을 때, **When** 페이지가 로드되면, **Then** B2C 프리미엄 브랜드와 B2B 전문 브랜드가 구분되어 카드 형태로 표시된다.
2. **Given** 방문자가 특정 브랜드 카드를 확인할 때, **When** 카드를 읽으면, **Then** 브랜드명, 슬로건, 대표 제품 카테고리, 간략한 소개가 표시된다.

---

### User Story 4 - 지속 가능성 스토리 탐색 (Priority: P3)

방문자가 `/sustainability` 페이지에서 SFood의 환경·사회적 책임 활동과 지속 가능한 식품 생태계 구축 비전을 확인할 수 있다.

**Why this priority**: ESG 관심도 증가로 브랜드 이미지 강화와 투자자·파트너 신뢰에 기여하지만, 직접적인 판매 전환 영향은 제한적이다.

**Independent Test**: `/sustainability` 단독 접속으로 지속 가능성 주요 활동 3개 이상과 미래 비전 문구가 표시되는지 확인 가능하다.

**Acceptance Scenarios**:

1. **Given** 방문자가 `/sustainability`에 접속했을 때, **When** 페이지가 로드되면, **Then** 환경·사회·식품 안전 등 지속 가능성 영역별 활동 내용이 표시된다.
2. **Given** 방문자가 페이지를 탐색할 때, **When** 전체 내용을 읽으면, **Then** SFood의 2030 지속 가능성 목표 또는 미래 비전이 명시되어 있다.

---

### User Story 5 - 인재 채용 정보 탐색 (Priority: P3)

구직자가 `/talent` 페이지에서 SFood의 인재상, 복리후생, 채용 프로세스를 확인하고 지원 의향을 높일 수 있다.

**Why this priority**: 우수 인재 확보를 위한 채용 페이지는 장기적으로 중요하나, 브랜드 사이트 초기 런칭에서는 보조적 역할이다.

**Independent Test**: `/talent` 단독 접속으로 인재상·채용 프로세스(단계별)·복리후생이 표시되는지 확인 가능하다.

**Acceptance Scenarios**:

1. **Given** 구직자가 `/talent`에 접속했을 때, **When** 페이지가 로드되면, **Then** SFood가 원하는 인재상, 핵심 역량 키워드가 표시된다.
2. **Given** 구직자가 채용 프로세스 섹션을 볼 때, **When** 해당 섹션을 확인하면, **Then** 서류 접수 → 서류 심사 → 면접 → 최종 합격의 단계별 프로세스가 시각적으로 표시된다.

---

### User Story 6 - 고객 지원 및 소통 채널 이용 (Priority: P3)

방문자가 `/support` 하위 페이지(공지사항·회사소식·FAQ)에서 최신 공지와 자주 묻는 질문을 확인하고 브랜드와 소통할 수 있다.

**Why this priority**: 신뢰 유지와 고객 서비스 기반을 제공하지만, 초기 런칭에서는 정적 콘텐츠로 구성되어 우선순위가 낮다.

**Independent Test**: `/support/notice`, `/support/news`, `/support/faq` 각각 단독 접속으로 해당 콘텐츠가 정상 표시되는지 확인 가능하다.

**Acceptance Scenarios**:

1. **Given** 방문자가 `/support/notice`에 접속했을 때, **When** 페이지가 로드되면, **Then** 공지사항 목록(제목·날짜)이 표시된다.
2. **Given** 방문자가 `/support/faq`에 접속했을 때, **When** 질문 항목을 클릭하면, **Then** 해당 답변이 펼쳐진다(아코디언 형태).
3. **Given** 방문자가 `/support/news`에 접속했을 때, **When** 페이지가 로드되면, **Then** 회사 소식 카드(제목·날짜·요약)가 표시된다.

---

### Edge Cases

- 존재하지 않는 URL(`/support/xyz` 등) 접속 시 SPA 클라이언트 라우터가 처리하는 404 안내 페이지(또는 메인 리다이렉트)를 제공해야 한다.
- 모바일 소형 화면(320px 이하)에서도 내비게이션 메뉴와 핵심 콘텐츠가 정상적으로 표시되어야 한다.
- 긴 텍스트 콘텐츠가 있을 때 레이아웃이 깨지지 않아야 한다.
- JavaScript 비활성화 환경에서도 기본 콘텐츠(텍스트·이미지)가 표시되어야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 클라이언트 사이드 라우팅(SPA) 방식으로 8개 경로(`/`, `/about`, `/sustainability`, `/brands`, `/talent`, `/support/notice`, `/support/news`, `/support/faq`)를 각각 독립적으로 제공해야 한다.
- **FR-002**: 모든 페이지는 일관된 글로벌 내비게이션(헤더·푸터)을 포함해야 하며, 현재 활성 메뉴 항목이 시각적으로 구분되어야 한다. 헤더는 화면 상단에 고정(sticky)되며, 페이지 최상단에서는 배경이 반투명(약 90% 불투명)하고 스크롤 시 브랜드 컬러 배경으로 완전 불투명 전환된다. 모바일 화면에서는 햄버거 버튼 클릭 시 오버레이/드로어 형태로 전체 메뉴가 펼쳐져야 한다.
- **FR-003**: 내비게이션은 페이지 URL에 맞춰 Home / 회사소개 / 지속가능성 / 브랜드 / 인재채용 / 고객지원 구조로 구성되어야 하며, "고객지원" 항목은 데스크톱에서 드롭다운 서브메뉴(공지사항·회사소식·FAQ)를, 모바일 드로어에서는 아코디언 서브메뉴를 제공해야 한다.
- **FR-004**: 시스템은 외부 API 없이 정적 가상 콘텐츠만으로 모든 페이지를 완성해야 한다.
- **FR-005**: 모든 텍스트 콘텐츠는 SFood의 브랜드 톤(밝고, 힘차고, 일관성 있는)에 맞춰 작성되어야 한다.
- **FR-006**: 메인 페이지는 히어로 섹션, 핵심 수치/수상 강조 섹션, 브랜드 소개 섹션, 지속 가능성 한줄 소개, CTA(Call-to-Action) 섹션을 포함해야 한다.
- **FR-007**: `/brands` 페이지는 B2C 브랜드 2개와 B2B 브랜드 2개(총 4개)를 시각적으로 구분하여 각 브랜드의 정체성과 대표 제품을 소개해야 한다.
- **FR-008**: `/talent` 페이지는 인재상, 채용 프로세스(단계별 시각화), 복리후생 섹션을 포함해야 한다.
- **FR-009**: `/support/faq` 페이지는 아코디언(펼침/접힘) 형태의 FAQ 목록을 제공해야 하며, 카테고리 필터 탭(전체·제품·주문·회사 등)을 포함해야 한다.
- **FR-010**: 사이트는 모바일·태블릿·데스크톱 화면 크기에 대응하는 반응형 레이아웃을 제공해야 한다.
- **FR-011**: 브랜드 정체성을 표현하기 위해 SFood 고유 컬러 팔레트(레드 계열 강조색 포함 커스텀 색상 변수)를 적용해야 한다.
- **FR-012**: 품질 인증(FSSC 22000, HACCP, DLG 수상) 정보는 `/about` 페이지에서 시각적 배지 또는 아이콘과 함께 표시되어야 한다.

### Key Entities

- **브랜드 (Brand)**: 브랜드명, 유형(B2B/B2C), 슬로건, 대표 제품 카테고리, 소개 문구 — B2C 2개(존쿡 델리미트 등 프리미엄 소비자 브랜드), B2B 2개(식품 서비스·유통 전문 브랜드)
- **공지사항 (Notice)**: 제목, 작성일, 내용 요약, 카테고리
- **회사소식 (News)**: 제목, 작성일, 요약, 대표 이미지 설명
- **FAQ 항목**: 질문, 답변, 카테고리(제품/주문/회사 등)
- **채용 프로세스 단계**: 단계명, 설명, 순서
- **지속 가능성 활동 (Sustainability)**: 영역(환경/사회/식품 안전), 활동명, 설명, 수치 목표

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 모든 8개 페이지가 URL 직접 접속 시 오류 없이 콘텐츠를 표시한다.
- **SC-002**: 방문자가 메인 페이지에서 원하는 서브 페이지로 내비게이션을 통해 3번 이내의 클릭으로 도달할 수 있다.
- **SC-003**: 모바일(375px), 태블릿(768px), 데스크톱(1280px) 화면 너비에서 레이아웃 깨짐 없이 모든 콘텐츠가 표시된다.
- **SC-004**: 각 페이지의 모든 섹션에 가상 콘텐츠가 빠짐없이 채워져 있고, 플레이스홀더(Lorem Ipsum 등) 텍스트가 남아 있지 않다.
- **SC-005**: SFood 고유 컬러 팔레트가 일관되게 적용되어 브랜드 아이덴티티를 시각적으로 전달한다.
- **SC-006**: `/support/faq` 페이지의 FAQ 항목이 클릭 시 정상적으로 펼쳐지고 접힌다.
- **SC-007**: 내비게이션의 모든 링크가 정확한 페이지로 연결된다(404 오류 없음).

## Assumptions

- 이 사이트는 클라이언트 사이드 라우팅(SPA, React Router 기반) 구조로 제작되며, 외부 백엔드 API나 데이터베이스 연동 없이 하드코딩된 가상 콘텐츠를 사용한다.
- 작업 폴더는 `/app/sfood-brand` (프로젝트 루트 기준)이며, 기존 `packages/web`과 유사한 구조로 구성된다.
- UI 컴포넌트 및 레이아웃 패턴은 `packages/ui`의 Storybook에 등록된 컴포넌트를 참고하여 구성한다.
- 브랜드 컬러로 딥레드(`#B01020` 계열)를 주 강조색으로, 오프화이트(`#FAF8F5`)를 배경색으로 사용한다.
- 폰트는 프로젝트 스택에 포함된 Pretendard(한국어) 및 Noto Sans KR을 우선 사용한다.
- 실제 제품 이미지나 로고는 없으므로, 색상 블록·SVG 플레이스홀더·이모지를 활용하여 시각적 완성도를 유지한다.
- 모든 콘텐츠(회사명·수상 내역·채용 공고 등)는 가상의 프리미엄 소시지 브랜드를 전제로 작성된 가상 데이터다.
- 다국어(한/영) 전환 기능은 v1 범위에서 제외한다.
