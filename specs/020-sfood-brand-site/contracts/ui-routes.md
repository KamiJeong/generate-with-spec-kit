# UI 라우트 계약: SFood 브랜드 사이트

## 목적

이 계약은 `apps/sfood`가 노출해야 하는 공개 URL, 페이지 목적, 필수 콘텐츠, 금지 동작을 정의한다. 첫 버전은 API 계약이 없으므로 사용자에게 보이는 UI 라우트가 주요 외부 인터페이스다.

## 공통 계약

- 모든 라우트는 SFood/에쓰푸드 브랜드 식별 요소, 공통 헤더, 공통 푸터를 표시한다.
- 모든 라우트는 기본 내비게이션에서 접근 가능하거나 지원 내비게이션에서 접근 가능해야 한다.
- 현재 라우트의 내비게이션 항목은 `aria-current="page"` 또는 동등한 접근성 표시를 제공해야 한다.
- 모든 페이지 제목은 문서 구조상 최상위 제목으로 식별 가능해야 한다.
- 콘텐츠는 한국어를 기본으로 작성한다.
- 정확한 인증/수상 명칭(`FSSC 22000`, `HACCP`, `DLG`)은 포함하지 않는다.

## 라우트 목록

| 경로 | 페이지 | 필수 콘텐츠 | 금지 동작 |
|------|--------|-------------|-----------|
| `/` | 메인 | 미션, 육가공 전문성, Meal Solution 확장, B2B/B2C 구조, 품질 신뢰 요소, 주요 페이지 이동 경로 | 구매, 문의 제출, 로그인 요구 |
| `/about` | 회사소개 | 회사 미션, 사업 영역, 제품 전문성, 품질 접근, 밝고 자신감 있는 회사 소개 | 정확한 인증/수상 명칭 사용 |
| `/sustainability` | 지속 가능성 | 책임 있는 원료, 효율적 생산, 식품 생태계, 미래 식문화 등 최소 3개 주제 | 검증되지 않은 법적 ESG 주장 |
| `/brands` | 브랜드 소개 | consumer 라인과 business 라인, 대표 제품군, 브랜드 포지셔닝 | 외부 구매 링크 필수화 |
| `/talent` | 인재 채용 | 인재상, 일하는 가치, 순서 있는 채용 프로세스 | 실제 지원서 제출 또는 개인정보 입력 |
| `/support/notice` | 공지사항 | 목록형 공지 샘플 3개 이상, 제목, 날짜형 라벨, 요약 | 공지 상세 페이지, 상세 링크 |
| `/support/news` | 회사소식 | 목록형 소식 샘플 3개 이상, 제목, 날짜형 라벨, 요약, 브랜드 관련 주제 | 소식 상세 페이지, 상세 링크 |
| `/support/faq` | FAQ | 브랜드, 제품, 품질, 구매/제휴, 채용 질문과 답변 | 검색 또는 서버 기반 필터 요구 |

## 상호작용 계약

- 기본 내비게이션 클릭은 브라우저 주소를 해당 경로로 변경하고 해당 페이지 콘텐츠를 표시한다.
- 지원 내비게이션은 `/support/notice`, `/support/news`, `/support/faq` 사이를 이동시킨다.
- FAQ 항목은 키보드로 열고 닫을 수 있어야 한다.
- 존재하지 않는 경로는 공통 레이아웃 안에서 찾을 수 없는 페이지 안내와 홈 이동 경로를 제공한다.

## 테스트 기대치

- 모든 계약 라우트가 렌더링 테스트에서 제목과 핵심 콘텐츠를 표시한다.
- 모든 내비게이션 `href`가 라우트 목록에 존재한다.
- 공지사항과 회사소식에는 상세 페이지를 암시하는 링크 또는 라벨이 없다.
- 금지된 인증/수상 명칭이 콘텐츠 데이터와 렌더링 결과에 없다.
- FAQ는 다섯 필수 카테고리를 모두 포함한다.
